Why use strToColor?

In many user interfaces a programmer needs a quick way of colourising entries in a list. Imagine a list of 50 blog posts on a page, authored by 10 different authors. Each of the 10 authors has 5 blog posts each, totaling 50 blog posts in the list. In this example we could use strToColour to colourise the blog post background, depending on the author name.

The colors are too random! (using transforms)

This procedure outlined above will give colors that are from any part of the spectrum, some bright some dark... This may not be suitable for all use cases (for example, the string "maybe" creates a colour that means you cant read the black text), so a color model is included so that you can do some basic transforms and get more harmoginised colors. 

strToColor.colorize('{{ teststring }}',{
	transform:[
		{setSaturation: 1},
		{setLightness: 0.6}
	]
});

I dont like the colours this method selected for my set of strings

You can switch the mode so that strToColor uses, so that different digits of the crc are used. There are 28 modes, numbered 0 to 27.
Each mode removes 2 digits from the 8 available.
mode 0 removes the last two digits of the crc.
mode 9 removes the middle two digits of the crc.
mode 27 removes the first two digits of the crc.
The modes option is really only useful if you dont like a particular set of colors for a particular set of words, or if websites dont want to use the same color schemes.
